# 04. 핸즈온: 지저분한 스크립트를 클린 코드로 리팩토링

이 실습에서는 나쁜 예제 코드를 점진적으로 리팩토링하여 **작고 명확한 함수**, **단일 책임 클래스**로 개선합니다.

## 준비
- `examples/bad_script.py`를 확인합니다.
- 목표: 기능은 동일, 구조는 더 명확하고 테스트 가능하도록 변경.

## 단계별 체크리스트
1. 함수 추출: 정제/필터/출력 역할 분리
2. 의미있는 이름 적용: 변수/함수/클래스 이름 개선
3. 가드절로 중첩 제거
4. 의존성 주입: 외부 자원(예: 출력/저장)을 주입 가능하게
5. 테스트 코드 작성(선택): 작은 함수 단위로 검증

## 완료 기준
- 각 함수/클래스가 단 하나의 책임만 가진다.
- 메인 로직은 작은 구성 요소들을 조합하여 문제를 해결한다.
- 변경이 쉬운 구조(새로운 필터 추가 등)를 갖춘다.

**examples 참조**
- `bad_script.py`: 초기 스파게티 코드
- `refactored.py`: 함수 분리, 책임 명확화 버전
