# 03. GitHub Issues: 할 일과 버그를 체계적으로 관리

## 핵심 사용법
- 이슈 생성: 제목/본문에 목적과 재현 절차를 명확히
- 라벨: `bug`, `feature`, `documentation` 등
- 마일스톤: 릴리스 목표와 일정을 연결
- Assignees: 담당자 지정

## 템플릿 활용
`.github/ISSUE_TEMPLATE/`에 템플릿을 두면 이슈 생성이 표준화됩니다.
- `bug_report.md`: 재현 단계, 기대/실제 결과, 환경정보
- `feature_request.md`: 배경, 요구사항, 수용 기준
