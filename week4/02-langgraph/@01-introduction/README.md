# LangGraph 소개

## LangGraph란?

LangGraph는 **상태 기반 에이전트 오케스트레이션 프레임워크**입니다. Klarna, Replit, Elastic 등 많은 기업들이 사용하고 있으며, 장기 실행되고 상태를 유지하는 에이전트를 구축, 관리, 배포하기 위한 저수준 프레임워크입니다.

## LangGraph의 특징

### 저수준 프레임워크

LangGraph는 **매우 저수준**이며, 에이전트 오케스트레이션에만 집중합니다. 프롬프트나 아키텍처를 추상화하지 않으며, 개발자가 완전한 제어권을 가질 수 있습니다.

### LangChain과의 관계

- **LangChain**: 높은 수준의 추상화, 빠른 프로토타이핑, 사전 구축된 에이전트 아키텍처
- **LangGraph**: 저수준 오케스트레이션, 완전한 커스터마이징, 복잡한 워크플로우 제어

**중요**: LangChain 에이전트는 내부적으로 LangGraph를 사용합니다. 하지만 기본적인 LangChain 에이전트를 사용할 때는 LangGraph를 직접 알아야 할 필요는 없습니다.

## LangGraph를 사용하는 경우

### LangGraph를 선택해야 하는 경우

- **결정론적 워크플로우와 에이전트 워크플로우의 조합**이 필요할 때
- **높은 수준의 커스터마이징**이 필요할 때
- **지연 시간을 세밀하게 제어**해야 할 때
- **복잡한 상태 관리**가 필요할 때
- **장기 실행되는 에이전트**를 구축해야 할 때

### LangChain을 선택해야 하는 경우

- 에이전트를 빠르게 시작하고 싶을 때
- 일반적인 LLM 및 도구 호출 루프를 위한 사전 구축된 아키텍처가 필요할 때
- 높은 수준의 추상화를 원할 때

## LangGraph의 핵심 개념

LangGraph는 에이전트 워크플로우를 **그래프**로 모델링합니다. 그래프는 다음 세 가지 핵심 구성 요소로 이루어져 있습니다:

1. **State (상태)**: 애플리케이션의 현재 스냅샷을 나타내는 공유 데이터 구조
2. **Nodes (노드)**: 에이전트의 로직을 인코딩하는 함수들
3. **Edges (엣지)**: 현재 상태를 기반으로 다음에 실행할 노드를 결정하는 함수들

## LangGraph의 주요 기능

### 1. Durable Execution (지속 가능한 실행)

장애가 발생해도 에이전트가 지속될 수 있으며, 오랜 시간 동안 실행될 수 있고, 중단된 지점부터 재개할 수 있습니다.

### 2. Human-in-the-Loop (사람 개입)

에이전트 상태를 언제든지 검사하고 수정할 수 있으며, 사람의 감독을 통합할 수 있습니다.

### 3. Comprehensive Memory (포괄적인 메모리)

단기 작업 메모리와 세션 간 장기 메모리를 모두 가진 상태 기반 에이전트를 만들 수 있습니다.

### 4. Debugging with LangSmith

실행 경로를 추적하고, 상태 전환을 캡처하며, 상세한 런타임 메트릭을 제공하는 시각화 도구를 사용할 수 있습니다.

### 5. Production-ready Deployment

상태 기반 장기 실행 워크플로우의 고유한 도전 과제를 처리하도록 설계된 확장 가능한 인프라로 정교한 에이전트 시스템을 배포할 수 있습니다.
