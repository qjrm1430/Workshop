# 명세(Specification)란 무엇인가?

## 명세의 정의

명세(Specification)는 **시스템이나 컴포넌트가 무엇을 해야 하는지, 어떻게 동작해야 하는지를 기술한 문서**입니다. 명세는 코드를 작성하기 전에 작성되며, 개발자와 AI 어시스턴트 모두가 이해할 수 있는 형태로 작성됩니다.

## 명세의 종류

### 1. 기능 명세 (Functional Specification)

시스템이 **무엇을 해야 하는지**를 설명합니다.

예시:
```
함수: calculate_total_price

목적: 장바구니의 모든 상품 가격을 합산하여 총액을 계산합니다.

입력:
  - items: List[Item] - 장바구니에 담긴 상품 목록
  - discount: float (선택) - 할인율 (0.0 ~ 1.0)

출력:
  - float - 총 가격 (할인 적용 후)

동작:
  1. 모든 상품의 가격을 합산
  2. 할인율이 제공되면 할인 적용
  3. 최종 가격 반환

예외:
  - items가 비어있으면 ValueError 발생
  - discount가 0.0 미만이거나 1.0 초과면 ValueError 발생

예제:
  items = [Item(price=1000), Item(price=2000)]
  total = calculate_total_price(items, discount=0.1)
  # 결과: 2700.0 (3000 * 0.9)
```

### 2. 기술 명세 (Technical Specification)

시스템이 **어떻게 구현되어야 하는지**를 설명합니다.

예시:
```
구현 요구사항:
  - Python 3.10 이상 사용
  - 타입 힌트 필수
  - 비동기 처리 지원
  - 로깅 포함
  - 단위 테스트 작성
```

### 3. API 명세 (API Specification)

API의 엔드포인트, 요청/응답 형식을 설명합니다.

예시:
```
POST /api/users

요청:
  {
    "name": "홍길동",
    "email": "hong@example.com"
  }

응답:
  {
    "id": 123,
    "name": "홍길동",
    "email": "hong@example.com",
    "created_at": "2024-01-01T00:00:00Z"
  }

에러:
  - 400: 잘못된 요청
  - 409: 이메일 중복
```

## 좋은 명세의 특징

### 1. 명확성 (Clarity)

명세는 모호함 없이 명확해야 합니다. "빠르게" 같은 표현보다는 "1초 이내"처럼 구체적인 수치를 사용합니다.

### 2. 완전성 (Completeness)

모든 경우를 다루어야 합니다. 정상 케이스뿐만 아니라 예외 케이스도 포함해야 합니다.

### 3. 검증 가능성 (Verifiable)

명세를 바탕으로 테스트를 작성할 수 있어야 합니다. "사용자 친화적"보다는 "에러 메시지는 한국어로 표시"처럼 검증 가능한 형태로 작성합니다.

### 4. 일관성 (Consistency)

명세 전체에서 용어와 형식이 일관되어야 합니다.

## 명세 작성 팁

### 1. 구조화된 형식 사용

- 목적
- 입력/출력
- 동작
- 예외
- 예제

### 2. 구체적인 예제 포함

추상적인 설명보다는 구체적인 예제가 더 이해하기 쉽습니다.

### 3. 엣지 케이스 고려

- 빈 입력
- 최대/최소 값
- null/None 처리
- 동시성 문제

### 4. AI 어시스턴트를 염두에 두고 작성

AI 어시스턴트가 이해하기 쉬운 명확한 언어를 사용합니다.

## 명세 작성 예제

### 나쁜 명세 예제

```
함수를 만들어서 사용자 데이터를 처리해주세요.
```

### 좋은 명세 예제

```
함수: process_user_data

목적: 사용자 데이터를 검증하고 정규화합니다.

입력:
  - user_data: dict - 사용자 데이터
    {
      "name": str,      # 필수, 1-50자
      "email": str,     # 필수, 유효한 이메일 형식
      "age": int        # 선택, 0-150
    }

출력:
  - dict - 정규화된 사용자 데이터
    {
      "name": str,      # 앞뒤 공백 제거, 첫 글자 대문자
      "email": str,     # 소문자로 변환
      "age": int | None
    }

동작:
  1. name 검증: 필수, 1-50자, 앞뒤 공백 제거 후 첫 글자 대문자
  2. email 검증: 필수, 유효한 이메일 형식, 소문자로 변환
  3. age 검증: 선택, 0-150 범위

예외:
  - name이 없거나 50자 초과: ValueError("이름은 1-50자여야 합니다")
  - email이 유효하지 않음: ValueError("유효한 이메일 주소를 입력하세요")
  - age가 범위를 벗어남: ValueError("나이는 0-150 사이여야 합니다")

예제:
  입력:
    {
      "name": "  홍길동  ",
      "email": "Hong@Example.COM",
      "age": 25
    }
  
  출력:
    {
      "name": "홍길동",
      "email": "hong@example.com",
      "age": 25
    }
```
