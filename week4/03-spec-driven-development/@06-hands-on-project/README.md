# 실습 프로젝트: Spec-driven Development로 기능 구현

이 실습 프로젝트에서는 Spec-driven Development 방법론을 사용하여 실제 기능을 구현해봅니다.

## 프로젝트 목표

Spec-driven Development의 전체 워크플로우를 경험합니다:
1. 요구사항 분석
2. 명세 작성
3. AI 어시스턴트를 활용한 코드 생성
4. 코드 검증 및 테스트
5. 반복적 개선

## 프로젝트 주제

**작업 관리 시스템의 핵심 기능 구현**

다음 기능들을 Spec-driven Development로 구현합니다:
- 작업 생성
- 작업 조회
- 작업 수정
- 작업 삭제
- 작업 상태 변경

## 단계별 실습

### 1단계: 요구사항 분석

작업 관리 시스템의 요구사항을 분석하고 정리합니다.

**과제**:
- 사용자 스토리 작성
- 기능 목록 정리
- 데이터 모델 정의

### 2단계: 명세 작성

각 기능에 대한 상세 명세를 작성합니다.

**과제**:
- 작업 생성 명세 작성
- 작업 조회 명세 작성
- 작업 수정 명세 작성
- 작업 삭제 명세 작성
- 작업 상태 변경 명세 작성

**명세 템플릿 사용**:
```
기능: [기능 이름]

목적: [무엇을 하는가]

입력:
  - [파라미터명]: [타입] - [설명]

출력:
  - [반환 타입] - [설명]

동작:
  1. [단계 1]
  2. [단계 2]
  3. [단계 3]

예외:
  - [예외 상황]: [처리 방법]

예제:
  [구체적인 사용 예제]
```

### 3단계: 명세 검토

작성한 명세를 검토하고 개선합니다.

**검토 항목**:
- 명확성
- 완전성
- 일관성
- 검증 가능성

### 4단계: 코드 생성

AI 어시스턴트(Claude Code 등)를 사용하여 명세를 바탕으로 코드를 생성합니다.

**활동**:
- 명세를 AI 어시스턴트에 제공
- 생성된 코드 검토
- 필요한 경우 명세 수정 및 재생성

### 5단계: 코드 검증

생성된 코드가 명세를 충족하는지 검증합니다.

**활동**:
- 코드 리뷰
- 수동 테스트
- 정적 분석

### 6단계: 테스트 작성

명세를 바탕으로 테스트를 작성합니다.

**테스트 종류**:
- 단위 테스트
- 통합 테스트
- 엣지 케이스 테스트

### 7단계: 반복적 개선

명세와 코드를 반복적으로 개선합니다.

**활동**:
- 피드백 수집
- 명세 수정
- 코드 개선
- 테스트 추가

## 실습 과제

1. **기본 기능 구현**: 위에서 설명한 5가지 기능을 모두 구현하세요.

2. **명세 품질 향상**: 초기 명세를 검토하고 개선하세요.

3. **테스트 커버리지**: 테스트 커버리지를 80% 이상 달성하세요.

4. **문서화**: 생성된 코드에 적절한 문서화를 추가하세요.

5. **확장**: 새로운 기능(예: 작업 검색, 작업 필터링)을 추가하세요.

## 평가 기준

- 명세의 명확성과 완전성
- 코드의 품질과 명세 준수도
- 테스트 커버리지
- 문서화 품질
- 반복적 개선 과정
